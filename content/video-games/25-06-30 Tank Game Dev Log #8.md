# A Shift into RTS
Over the past month i've been coding a turn-based strategy game where you have a timeline and you get to set all your unit's actions for the next 5 seconds. There will be elements from BAR, Broken Arrow, and some other strategy titles.

I don't have a clear picture in mind of what the end product is gonna turn out to be yet. But I can sense that it is going to be pretty complex, with multiple systems I have to make.

# It's Sunset

While slowly coding the game, the code base gets larger and the code starts to get messy. I think it's because I didn't do a proper design of the game + it needs many complex stuff. I also have a feeling that the end result might not be the game that I want to play the most. 

So I decided to stop it's development.

I would not say that it is a failure, because I did intend it to be a "**tech study** of RTS unit navigation system". Which I think i did learn a bit about that and also other things.

I even made this cute **Frog Soldier** 🐸 with blender. I must say 3D modelling is pretty fun too.

![Frog Soldier](</image/video-games/TankGame/Frog Soldier render02.png#small>)

My partner told me that when you do something, there's no losing, only win or learn 🌞.
